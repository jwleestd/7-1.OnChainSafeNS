# AI Rule 파일 MVP 정합성 분석 및 개선 제안

> **목표:** PRD v0.5 / SRS v0.8에서 정의한 **Phase-0 Actionable MVP** 기술 스택·아키텍처·설계 원칙에 맞도록 프로젝트의 AI Rule 파일(AGENTS.md, CLAUDE.md, .agents/rules/*, .cursor/rules/*)을 전면 정비한다.

## 현황 요약

현재 규칙 파일들은 **이전 프로젝트(사업계획서 작성 도구)**의 템플릿이 그대로 남아 있으며, PRD/SRS에서 정의한 **온체인 사기 방지 플랫폼**의 실제 기술 스택·아키텍처와 **전혀 맞지 않는 상태**입니다.

| 항목 | 현재 규칙 파일 | PRD/SRS 정의 (MVP) |
|---|---|---|
| 프로젝트명 | 미기재 또는 "사업계획서 도구" | **온체인 사기 방지 플랫폼 (On-Chain Fraud Shield)** |
| Frontend | Vite + React + TypeScript (SPA) | **Next.js App Router (풀스택)** |
| Backend | Spring Boot 4.0 + Java 21 | **Next.js Route Handlers (TypeScript)** |
| AI Engine | Python + FastAPI + LangChain | **Phase-3에서 Vercel AI SDK + Gemini** |
| Database | MySQL 8.x + JPA | **Supabase PostgreSQL + Prisma ORM** |
| Architecture | Micro-Service Ready | **단일 풀스택 모노리스 (2계층)** |
| Auth | OAuth2 (Google, KakaoTalk) | **Admin PW + 이메일 인증 (JWT/RBAC 없음)** |
| Messaging | Apache Kafka | **없음 (MVP 범위 밖)** |
| Caching | Redis | **없음 (MVP 범위 밖)** |
| Styling | Tailwind CSS (Cursor), 미지정(Agents) | **Tailwind CSS + shadcn/ui** |
| Deploy | Docker | **Vercel (Git Push 자동 배포)** |

> [!CAUTION]
> **현재 모든 규칙 파일이 완전히 다른 프로젝트의 기술 스택을 지시하고 있어, AI가 코드를 생성할 때 Spring Boot/Java/MySQL 기반으로 잘못 생성할 위험이 매우 높습니다.**
